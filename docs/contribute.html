Contribute

Submit a story, media item, or archival record.

Use this page to send contributions to the WITH archive system. Submission metadata is saved in MongoDB, while uploaded files are stored in the GitHub repository and reviewed through the dashboard before becoming publicly visible.

Open form Review dashboard
Story Narrative writing, reflections, oral histories
Media Images, posters, scans, event visuals
Record Newsletters, files, administrative documents

What happens next

  • Your form is sent to the backend API.
  • The contribution metadata is saved in MongoDB.
  • Any uploaded file is stored in the GitHub repository by file type and collection path.
  • The item appears in the dashboard for approval before it is shown on the public site.
Review flow Approved items can be displayed in the correct site category such as Story, Media, or Record.

Notes

Submission guidance

This version sends the contribution to the backend so the record can be stored in MongoDB and the file can be saved in GitHub for site use.

  • Use clear, descriptive titles.
  • Add tags for easier filtering later.
  • Choose the closest collection grouping.
  • Upload a file when the contribution includes a document, image, or scan.
  • Items remain pending until approved in the dashboard.
Backend reminder Set the API URL below to your deployed Railway backend, not localhost, when publishing the site on GitHub Pages.
